深入理解C语言

个人复习用，内容源于荣文戈老师的《C语言系统级编程》一课

基础

内存

  • bit（比特）：最小的存储单元，二进制的状态0/1
  • byte（字节）：若干个bit组成一个字节。
    • 一个byte至少8个bit，常见的1byte=8bit。
    • C语言中规定1byte=CHAR_BIT个bit，即sizeof(char)=1(B)
  • 内存：一系列byte顺序排列组成的存储结构
  • 32位机：用32位bit表示byte的编号，即byte共有$2^32$个，对应了存储空间为$2^32=4G$ (Byte)的存储空间大小

类型

类型(type)分为Object Type和Function Type，即对象类型和函数类型。

对象类型又包括完全对象类型和不完全对象类型

完全对象类型如：

  • int
  • short
  • long
  • float
  • double

不完全对象类型如：

  • union
  • struct
  • enum

判断一个式子，如int[5]，是否是对象类型，可以用sizeof()进行判断：
sizeof()只引导对象类型

sizeof

数组对象类型

C语言中只有一维数组：任何数组的类型都是array of T，其中T是数组元素的类型。

int a[2][3][4]，a标识的对象的类型是int[2][3][4]，属于数组类型，数组元素对象类型为int[3][4]

指针对象类型

指针对象类型属于非数组对象类型。

任何一种对象类型，都存在指向其的一个指针对象类型，反之亦然。(Reference Type <–> Pointer Type)

对象(object)

定义

objects are composed of contiguous sequences of one of more bytes

对象是内存中的一段连续的存储空间

属性

考察对象的属性：

  • Object Name（对象名字）：标识符
    • 当malloc分配得到对象时没有名字
  • Object Representation（对象表示）：对象对应在内存中由各bit组成的二进制串
  • Object Value：Object Type+Object Representation（对象类型+对象表示）

Object Value

对对象取值（Value)，得到这个值的内容和值的类型（内存视角，所得的Value为内存的表示值，Value Type为表示值的类型）：

<Value,Value Type>

区分Value Type和对象自身的类型Object Type，对应规则如下

  • Object Type为非数组对象类型
    • Value Type = Object Type
    • Object Representation按照规则转换成Value
  • Object Type为数组对象类型
    • Value Type = Pointer Type points to element Object Type
    • Value = head address of first element

例如：
int a[2];

  • a是一个对象，Object Type为int[2]，为数组对象类型,元素的对象类型为int
  • 对a取值，Value Type为指向元素类型的指针类型，即int*，Value为第一个元素的首地址

对象分配

对象分配（分配一段内存）的方式主要是以下两种：

  1. 变量声明
  2. malloc分配

变量声明

变量声明可以形式化为 Obj_T N

  • Obj_T : Object Type
  • N : Object identifier

如int a，其中a为标识符，在编译时存放在符号表中用于识别和定位这块内存（对象），int则规定了这块内存的类型和大小

对于内存，有两个概念：

  • 内存的变量类型 <-> 内存对应对象的类型Obj_T
  • 内存的表示值类型：对应上述的值(<Value,Value Type>)，内存的表示值是外界通过内存对应的对象/变量去观察这一块内存得到的结果

内存六元组

为刻画对象，提出内存六元组的逻辑模型。

  • Address：对象对应内存的第一个字节的编号
  • Object Type: 对象类型
  • Name: 变量名称
  • Size：内存大小(Byte)
  • Value：内存的表示值（01串）
  • Value Type: 表示值的类型

其中address由系统分配，确定后无法更改

涉及内存的常见操作有：
内存分配、内存赋值、内存释放、内存信息读取，在这里我们首先关注内存定位的过程

内存定位

内存定位方式有两种：

  1. 变量名/Identifier定位
  2. *运算符定位：对于表达式exp，其取值为<Value,Value_Type>，若Value_Type是一个指针类型，则可以使用*exp定位到Value对应字节编号开头的一段内存

*运算符定位

对于*运算符定位（如*p，p的obj_t为int*)，过程如下：

  1. 对p取值，得到<value,int*>
  2. 校验p的value type为有效指针类型
  3. 根据value定位到内存地址，该内存对应的obj_t为int
  4. 不妨令int temp = *p,temp和*p的作用完全一致

表达式(<value,Pointer Type>)做*运算得到的返回值为<obj_address=value,Reference Type>，*运算符相当于向系统说明exp获取到的值是一个有效address，可用于定位对象/内存。

对于指针变量p，p+n依然具备定位的功能：p取值value type为指针类型，n为常数，为了使得双方类型兼容，系统计算返回的value’=value+sizeof(p)*n，p表示p指针类型所指向的reference type，上述操作完成了地址的偏移，返回的value Type与p的value type保持一致。

*(exp+n)和exp[n]是一致的，所以int*p ; (p+1)[2]和p[3]是一致的

解析：

  1. n：<address=0x00FF3811,obj_t=int[2][3][4][5],value=0x00FF3811,Value_Type=int(*)[3][4][5]>
  2. &n: <value=n.value,Value_Type=pointer points to n.obj_t=int(*)[2][3][4][5]>
  3. &n+1: &n是一个指针类型，&n+1保留value type,<value=&n.value+1*sizeof(*(&n))= 0x00FF3811+1*sizeof(int[2][3][4][5])>
  4. n[0]: 相当于*n，定位得到对象<address=n.value,obj_t=int[3][4][5]>，取其表示值为<value=0x00FF3811,Value_Type=int(*)[4][5]>
  5. n[0]+1: n[0]的表示值如上，属于指针类型，n[0]+1的值为<value=0x00FF3811+1*sizeof(int[4][5]),int(*)[4][5]>
  6. n[0][0]: 相当于*(n[0])，定位得到对象<address=0x00FF3811,obj_t=int[4][5]>，取其表示值为<value=0x00FF3811,Value_Type=0x00FF3811,int(*)[5]>
  7. n[0][0]+1: n[0][0]的值如上，属于指针类型，n[0][0]+1的值为<value=0x00FF3811+1*sizeof(int[5]),value_type=int(*)[5]>
  8. n[0][0][0]: 等价于*(n[0][0])，定位得到对象<address=0x00FF3811,obj_type=int[5]>，取其表示值为<value=0x00FF3811,value_type=int*>
  9. n[0][0][0]+1: n[0][0][0]表示值已知，属于指针类型，则n[0][0][0]+1的值为<value=0x00FF3811+1*sizeof(int),value_type=int*>
  10. n[0][0][0][0]: *(n[0][0][0])，定位得到对象<address=0x00FF3811,obj_t=int>,值为<value=0,value_type=int>
  11. n[0][0][0][0]+1: <value:1, value_type=int>

变量名定位

  1. a是一个变量名，名称a及对象的首地址记录在符号表中
  2. a是一个左值（可定位），用a定位内存
  3. 计算等号右边的表达式，exp的值为<10,int>
  4. 类型匹配：a的六元组中的Value Type=int
  5. 内存赋值：系统按照a的Value Type将右值Value(10)转换为int型的32位01串并填入内存

内存操作

  1. （使用表达式）定位内存
    1. 变量名表达式定位内存
    2. *指针类型表达式定位内存
  2. 对内存的操作
    1. 获取内存首地址：&（exp)：要求exp为左值表达式（可以定位到内存），过程如下
      1. exp定位内存（对象）
      2. &操作：取对象的address和Obj_T
      3. 返回值为<address,Obj_T>
        例如：int a=10;&a得到<address,int*>
    2. 获得内存大小：sizeof(exp)
      1. exp的Value Type需要是对象类型
      2. 返回值类型为size_t
    3. 获得内存的表示值：表达式本身

表达式

左值（Lval)

左值本意不是等号左边的表达式，而是locator，可以定位内存/Object的表达式。

左值只有两种情况：identifier和*exp，exp的值为<value,pointer>

并不是所有左值都能放到等号左边，只有Modifiable lvalue才能放到左边，条件是lvalue所定位的内存：

  • 变量类型不是数组类型
  • 变量类型不是不完全类型
  • 不能被const修饰
  • 如果变量类型是struct/union，则成员的变量类型也不能被const修饰

小结

对于对象p，用内存六元组去刻画它的属性
*p:

  1. p取值<value,value_type>
  2. *运算，定位到对象<address=value,obj_type=reference type which is pointed by value_type>
  3. *p对象取值: <value,value_type>，此处的value和value_type依据Obj_type而定

&p:

  1. p定位到对象
  2. &运算，返回<value=address, pointer type points to obj_type>，此时返回值并不是一个lval
